Para empezar, es importante entender cómo Claude Code utiliza las variables de entorno para configurar su comportamiento. Por ejemplo, la variable `ANTHROPIC_BETAS` se utiliza para habilitar características beta de la API de Anthropic. Sin embargo, esta variable no está documentada en la página de variables de entorno de Claude Code, lo que puede causar confusión para los desarrolladores. Para solucionar este problema, se puede utilizar el comando `export ANTHROPIC_BETAS=` para establecer la variable de entorno antes de ejecutar Claude Code.

La configuración de Claude Code también se puede realizar mediante el archivo `CLAUDE.md`, que se utiliza como guía de inicio para el modelo de lenguaje. En este archivo, se pueden incluir descripciones del proyecto, explicaciones de los componentes principales y direcciones de los directorios. Por ejemplo, se puede incluir la siguiente estructura de directorios: `my-project/ ├── CLAUDE.md ├── package.json ├── src/ ├── docs/ └── scripts/`. Esto ayuda a Claude Code a entender la estructura del proyecto y a proporcionar sugerencias y completados más precisos.

Otra característica importante de Claude Code es su capacidad para trabajar con flujos de trabajo y tareas. La herramienta Scopecraft Command es una herramienta de línea de comandos y servidor MCP que ayuda a gestionar archivos de gestión de tareas basados en Markdown. Se puede utilizar el comando `scopecraft init` para inicializar un nuevo proyecto y `scopecraft run` para ejecutar las tareas definidas en el archivo de configuración. Esto permite a los desarrolladores automatizar tareas y flujos de trabajo de manera eficiente.
