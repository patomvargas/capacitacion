Apache Airflow 3.2: Mejoras y Best Practices

2026-04-28 1020 · Airflow

La versión 3.2 de Apache Airflow introduce cambios importantes en la arquitectura y el modelo de ejecución de tareas. Una de las mejoras más destacadas es la partición de activos (Asset partitioning) para la orquestación de pipelines granulares. Esto permite a los desarrolladores crear pipelines más complejos y escalables.

Para aprovechar esta función, se puede utilizar el comando `airflow db init` para inicializar la base de datos y luego crear un nuevo DAG con el comando `airflow dags create`. Luego, se puede definir la partición de activos en el archivo `airflow.cfg` utilizando la configuración `asset_partitioning_enabled = True`.

Otra mejora importante es la introducción de la función de deadline de alertas sincrónicas (synchronous deadline alert callbacks). Esto permite a los desarrolladores crear tareas que se ejecuten de manera sincrónica y reciban alertas en tiempo real. Para utilizar esta función, se puede agregar un sensor de deadline a un DAG utilizando el comando `airflow sensors add`.

La versión 3.2 también introduce mejoras en la autorización de tareas y la gestión de usuarios. Se puede utilizar el comando `airflow users create` para crear nuevos usuarios y asignarles permisos utilizando el comando `airflow roles add`. Luego, se puede definir la autorización de tareas en el archivo `airflow.cfg` utilizando la configuración `authorization_enabled = True`.

En cuanto a best practices, se recomienda utilizar la función de partición de activos para crear pipelines más escalables y complejos. También se recomienda utilizar la función de deadline de alertas sincrónicas para crear tareas que se ejecuten de manera sincrónica y reciban alertas en tiempo real. Finalmente, se recomienda utilizar la función de autorización de tareas para gestionar los permisos de los usuarios y proteger la seguridad de la plataforma.
