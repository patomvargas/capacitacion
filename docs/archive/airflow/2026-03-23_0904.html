Airflow 2.x: Mejores Prácticas y Nuevas Características

2026-03-23 0904 · Airflow

Airflow 2.x introduce varias novedades y mejoras en comparación con sus versiones anteriores. Una de las características más destacadas es el soporte para DeltaTriggerTimetable, que permite programar trabajos basados en deltas sin depender de intervalos de datos. Para utilizar esta característica, se puede crear un `DeltaTriggerTimetable` en el archivo `airflow.cfg` con el siguiente comando: `airflow config set core delta_trigger_timetable True`.

La versión 2.11.2 de Airflow también incluye mejoras en la gestión de proveedores, lo que facilita la integración con otros servicios y herramientas. Por ejemplo, se puede utilizar el comando `airflow providers list` para ver la lista de proveedores disponibles y `airflow providers install` para instalar nuevos proveedores.

Otra característica importante de Airflow 2.x es el soporte para XCom, que permite la comunicación entre tareas y la reutilización de resultados. Para utilizar XCom, se puede crear un `XComArg` en el código de la tarea y luego utilizar el método `xcom_pull` para recuperar el valor. Por ejemplo: `xcom_arg = XComArg(task_id='mi_tarea', key='mi_valor')` y luego `valor = xcom_pull(task_id='mi_tarea', key='mi_valor')`.

En cuanto a las mejores prácticas, es importante mencionar la importancia de utilizar DAGs bien estructurados y organizados. Un DAG bien diseñado debe tener una lógica clara y fácil de seguir, y debe incluir comentarios y documentación para facilitar su comprensión. También es importante utilizar operadores y sensores de manera efectiva para automatizar y controlar los flujos de trabajo.

Finalmente, es importante destacar la importancia de mantener actualizado Airflow y utilizar las últimas versiones de los proveedores y dependencias. Esto garantiza que se aprovechen las últimas características y mejoras de seguridad, y reduce el riesgo de vulnerabilidades y errores.

Para más información sobre Airflow y sus características, se puede consultar la documentación oficial y los recursos en línea, como la página de GitHub de Apache Airflow y los tutoriales y guías de Astronomer.
