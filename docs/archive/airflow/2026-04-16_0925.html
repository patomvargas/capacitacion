Airflow 3.2: Mejoras y Novedades

2026-04-16 0925 · Airflow

Apache Airflow 3.2 es la última versión del popular sistema de gestión de flujos de trabajo. Esta versión introduce varias mejoras y novedades, incluyendo la partición de activos, que permite una gestión más granular de los pipelines. Para aprovechar esta función, se puede utilizar el comando `airflow db init` para inicializar la base de datos y luego configurar la partición de activos en el archivo `airflow.cfg`.

La separación de tareas es otra de las mejoras importantes en Airflow 3.2. Esto permite una mayor flexibilidad y escalabilidad en la gestión de flujos de trabajo. Para separar tareas, se puede utilizar el comando `airflow tasks list` para ver la lista de tareas disponibles y luego utilizar el comando `airflow tasks run` para ejecutar una tarea específica.

La nueva interfaz de usuario de Airflow 3.2 es otra de las mejoras significativas. La interfaz de usuario ha sido reescrita en React y ofrece una experiencia de usuario más intuitiva y fácil de usar. Para acceder a la interfaz de usuario, se puede utilizar el comando `airflow webserver` para iniciar el servidor web y luego acceder a la dirección `http://localhost:8080` en el navegador.

En cuanto a la escalabilidad, Airflow 3.2 incluye varias mejoras, incluyendo la capacidad de ejecutar múltiples instancias del servidor web y la capacidad de utilizar bases de datos distribuidas. Para configurar la escalabilidad, se puede utilizar el comando `airflow config list` para ver la lista de configuraciones disponibles y luego utilizar el comando `airflow config set` para establecer una configuración específica.

En resumen, Airflow 3.2 es una versión importante que introduce varias mejoras y novedades en la gestión de flujos de trabajo. La partición de activos, la separación de tareas y la nueva interfaz de usuario son solo algunas de las mejoras que hacen de Airflow 3.2 una herramienta más poderosa y flexible para la gestión de flujos de trabajo.
