Ansible: Automatización de Infraestructura

2026-04-21 0944 · Ansible

Ansible es una herramienta de automatización de infraestructura que utiliza playbooks, roles, collections, vault, inventory y AWX/Tower para gestionar y configurar entornos de manera eficiente. Un playbook es un archivo YAML que define una serie de tareas que se deben ejecutar en un conjunto de hosts. Los roles son conjuntos predefinidos de tareas que se pueden reutilizar en diferentes playbooks. Las collections son conjuntos de módulos y roles que se pueden instalar y utilizar en los playbooks.

Para empezar a utilizar Ansible, es necesario instalar el software en el servidor de control y configurar el inventario de hosts. El inventario se puede definir en un archivo `hosts.ini` o `hosts.yaml` que contiene la lista de hosts y sus respectivas variables. Por ejemplo, se puede definir un inventario con los siguientes hosts: `web`, `db` y `lb`. Luego, se puede crear un playbook que utilice este inventario para configurar los hosts.

Un ejemplo de playbook que configura un servidor web Apache en un host `web` sería:

```yaml

---

- name: Configurar servidor web Apache

hosts: web

become: yes

tasks:

- name: Instalar Apache

apt:

name: apache2

state: present

- name: Configurar archivo de configuración de Apache

template:

src: templates/apache.conf.j2

dest: /etc/apache2/apache.conf

mode: '0644'

```

Este playbook utiliza el módulo `apt` para instalar Apache y el módulo `template` para configurar el archivo de configuración de Apache.

En cuanto a la seguridad, Ansible proporciona la herramienta `ansible-vault` para cifrar archivos y variables sensibles. Por ejemplo, se puede cifrar un archivo de configuración de base de datos con la siguiente comando: `ansible-vault encrypt db.conf`. Luego, se puede utilizar el archivo cifrado en un playbook con la siguiente tarea:

```yaml

- name: Configurar base de datos

template:

src: templates/db.conf.j2

dest: /etc/db.conf

mode: '0644'

vault: db.conf

```

Este playbook utiliza el archivo cifrado `db.conf` para configurar la base de datos.

En resumen, Ansible es una herramienta poderosa para la automatización de infraestructura que permite a los administradores de sistemas y DevOps engineers gestionar y configurar sus entornos de manera eficiente. Con playbooks, roles, collections, vault, inventory y AWX/Tower, Ansible proporciona una amplia gama de herramientas para automatizar tareas y asegurar la configuración de los entornos.
