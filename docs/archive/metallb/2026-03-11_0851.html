MetalLB: Balanceo de Carga en Kubernetes

2026-03-11 0851 · MetalLB

MetalLB es un sistema de balanceo de carga que se ejecuta en modo L2 o BGP, lo que permite la asignación de direcciones IP a servicios en un clúster de Kubernetes. Para configurar MetalLB en modo L2, se debe crear un objeto de configuración con el comando `kubectl apply -f metallb-config.yaml`, donde el archivo `metallb-config.yaml` contiene la configuración del sistema.

Un ejemplo de configuración en modo L2 es el siguiente:

```

apiVersion: metallb.io/v1beta1

kind: Config

metadata:

name: metallb-config

spec:

peers:

- peerAddress: 10.0.0.1

peerPort: 7946

```

Para configurar MetalLB en modo BGP, se debe crear un objeto de configuración similar, pero con la configuración de BGP.

Los pools de IP son una característica importante de MetalLB, ya que permiten la asignación de direcciones IP a servicios de manera dinámica. Para crear un pool de IP, se debe ejecutar el comando `kubectl apply -f ip-pool.yaml`, donde el archivo `ip-pool.yaml` contiene la configuración del pool de IP.

Un ejemplo de configuración de un pool de IP es el siguiente:

```

apiVersion: metallb.io/v1beta1

kind: IPAddressPool

metadata:

name: ip-pool

spec:

addresses:

- 10.0.0.10-10.0.0.20

```

La integración con Kubernetes es una de las características más importantes de MetalLB. Para integrar MetalLB con Kubernetes, se debe crear un objeto de servicio con el comando `kubectl apply -f service.yaml`, donde el archivo `service.yaml` contiene la configuración del servicio.

Un ejemplo de configuración de un servicio es el siguiente:

```

apiVersion: v1

kind: Service

metadata:

name: servicio

spec:

selector:

app: aplicacion

ports:

- name: http

port: 80

targetPort: 8080

type: LoadBalancer

```

El troubleshooting es una parte importante de la administración de MetalLB. Para solucionar problemas comunes, se pueden ejecutar comandos como `kubectl get pods -n metallb-system` para verificar el estado de los pods de MetalLB, o `kubectl logs -f -n metallb-system` para ver los logs de los pods de MetalLB.

En resumen, MetalLB es un sistema de balanceo de carga que se integra perfectamente con Kubernetes, y que permite la asignación de direcciones IP a servicios de manera dinámica. Con las configuraciones y comandos adecuados, se puede asegurar un funcionamiento óptimo del sistema en un entorno de producción.
