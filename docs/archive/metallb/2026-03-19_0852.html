MetalLB: Balanceo de Carga en K8s

2026-03-19 0852 · MetalLB

MetalLB es una herramienta de código abierto que permite a los administradores de sistemas implementar servicios de balanceo de carga en un clúster de Kubernetes. Una de las características más destacadas de MetalLB es su capacidad para funcionar en modo L2, lo que permite a los pods comunicarse directamente con los clientes sin necesidad de un router externo. Para configurar MetalLB en modo L2, se debe crear un objeto de configuración de tipo `MetalLB` con el modo `layer2` especificado, como se muestra en el siguiente ejemplo: `kubectl apply -f 'metalLB-config.yaml'` con contenido `apiVersion: metallb.io/v1beta1

kind: MetalLB

metadata:

name: metallb

spec:

addresses:

- 192.168.1.100-192.168.1.150

peers:

- peer:

ip: 192.168.1.1

port: 7946

myID: 1

mode: layer2`.

Para implementar MetalLB en modo BGP, se debe configurar un objeto de tipo `BGPAdvertisement` que especifique los anuncios BGP que se deben enviar a los routers vecinos. Por ejemplo: `kubectl apply -f 'bgp-adv.yaml'` con contenido `apiVersion: metallb.io/v1beta1

kind: BGPAdvertisement

metadata:

name: bgp-adv

spec:

peer:

ip: 192.168.1.1

port: 179

advertisement:

prefix: 192.168.1.100/24

nextHop: 192.168.1.100`.

Otra característica importante de MetalLB es la capacidad de configurar pools de direcciones IP, lo que permite a los administradores de sistemas asignar direcciones IP a los servicios de balanceo de carga de manera más eficiente. Para crear un pool de direcciones IP, se debe crear un objeto de tipo `IPPools` con la lista de direcciones IP que se deben incluir en el pool, como se muestra en el siguiente ejemplo: `kubectl apply -f 'ip-pool.yaml'` con contenido `apiVersion: metallb.io/v1beta1

kind: IPPool

metadata:

name: ip-pool

spec:

addresses:

- 192.168.1.100-192.168.1.150`.

En cuanto a la integración con Kubernetes, MetalLB se puede integrar con el sistema de orquestación de contenedores para proporcionar servicios de balanceo de carga a los pods. Para hacer esto, se debe crear un objeto de tipo `Service` con el tipo `LoadBalancer` especificado, como se muestra en el siguiente ejemplo: `kubectl apply -f 'service.yaml'` con contenido `apiVersion: v1

kind: Service

metadata:

name: my-service

spec:

selector:

app: my-app

ports:

- name: http

port: 80

targetPort: 8080

type: LoadBalancer`.

Para solucionar problemas comunes con MetalLB, se pueden utilizar herramientas como `kubectl` para verificar el estado de los objetos de configuración y los logs de los componentes de MetalLB. Por ejemplo, se puede utilizar el comando `kubectl get metallb` para verificar el estado del objeto de configuración de MetalLB, o `kubectl logs -f metallb-controller` para verificar los logs del controlador de MetalLB.
