MetalLB: Equilibrio de Carga en Kubernetes

2026-03-10 1212 · MetalLB

MetalLB es una herramienta de equilibrio de carga para Kubernetes que permite asignar direcciones IP reales a servicios de tipo LoadBalancer. Esto es especialmente útil en entornos de bare-metal, donde no se dispone de un equilibrio de carga integrado. Para configurar MetalLB, es necesario crear un objeto `IPAddressPool` que defina el rango de direcciones IP que se asignarán a los servicios.

```yaml

apiVersion: metallb.io/v1beta1

kind: IPAddressPool

metadata:

name: pool-local

spec:

autoAssign: false

addresses:

- 192.168.3.247-192.168.3.250

```

Una vez creado el objeto `IPAddressPool`, se puede configurar MetalLB para que utilice el modo L2 o BGP. El modo L2 es el más sencillo y permite asignar direcciones IP a servicios de tipo LoadBalancer sin necesidad de configurar un router BGP. Para configurar MetalLB en modo L2, se puede utilizar el siguiente comando: `kubectl apply -f https://raw.githubusercontent.com/metallb/metallb/v0.15.2/manifests/layer2.yaml`.

En cuanto a la integración con Kubernetes, MetalLB se puede configurar para que se integre con el sistema de orquestación de contenedores. Para hacer esto, se debe crear un objeto `Service` de tipo LoadBalancer y asignarle una dirección IP del rango definido en el objeto `IPAddressPool`. Por ejemplo:

```yaml

apiVersion: v1

kind: Service

metadata:

name: mi-servicio

spec:

selector:

app: mi-aplicacion

ports:

- name: http

port: 80

targetPort: 8080

type: LoadBalancer

```

Es importante mencionar que MetalLB también admite el modo BGP, que permite asignar direcciones IP a servicios de tipo LoadBalancer utilizando el protocolo BGP. Para configurar MetalLB en modo BGP, se debe crear un objeto `BGPAdvertisement` que defina la configuración del router BGP.

En resumen, MetalLB es una herramienta poderosa para equilibrio de carga en Kubernetes que permite asignar direcciones IP reales a servicios de tipo LoadBalancer. Su configuración y uso en modo L2 y BGP hacen que sea una herramienta versátil y escalable para entornos de bare-metal y cloud.

Para solucionar problemas comunes con MetalLB, se pueden utilizar herramientas como `kubectl` y `metallbctl` para verificar la configuración y el estado de los servicios. También es importante monitorear los logs de MetalLB para detectar cualquier problema o error.

Finalmente, es importante mencionar que MetalLB se puede integrar con otras herramientas y sistemas de orquestación de contenedores, como Talos Linux y Rancher. Para obtener más información sobre la configuración y el uso de MetalLB, se pueden consultar las fuentes oficiales y la documentación de la comunidad.
