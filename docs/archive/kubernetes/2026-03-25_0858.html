Novedades y Mejoras en Kubernetes

2026-03-25 0858 · Kubernetes

La última versión de Kubernetes, 1.35, se lanzó recientemente con varias mejoras y nuevas características. Una de las novedades más destacadas es la retirada de Ingress-NGINX, que se realizará en marzo de 2026. Esto significa que los administradores de clusters deben migrar a otras soluciones de ingreso, como `ingress-nginx` con `rewrite-target`.

Para migrar a `ingress-nginx` con `rewrite-target`, se puede utilizar el siguiente comando: `kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/nginx-0.30.0/deploy/static/mandatory.yaml`. Luego, se debe configurar el `ingress` con `rewrite-target` utilizando un archivo de configuración como el siguiente:

```yaml

apiVersion: networking.k8s.io/v1

kind: Ingress

metadata:

name: ejemplo

spec:

rules:

- host: ejemplo.com

http:

paths:

- path: /

pathType: Prefix

backend:

service:

name: ejemplo

port:

number: 80

ingressClassName: nginx

```

Otra novedad importante es la mejora en la seguridad de Kubernetes, con la resolución de varias vulnerabilidades, como la CVE-2026-3288. Para protegerse contra este tipo de vulnerabilidades, se recomienda utilizar `network policies` para restringir el tráfico entre pods y servicios.

Además, se han realizado mejoras en la gestión de almacenamiento, con la introducción de nuevas características como el soporte para `CSI` (Container Storage Interface). Esto permite a los administradores de clusters utilizar una variedad de soluciones de almacenamiento, como `Ceph` o `Gluster`, para almacenar datos de los pods.

Para implementar `CSI` en un cluster de Kubernetes, se puede utilizar el siguiente comando: `kubectl apply -f https://raw.githubusercontent.com/kubernetes/csi-plugin/master/deploy/kubernetes/csi-plugin.yaml`. Luego, se debe configurar el `storage class` para utilizar la solución de almacenamiento deseada.

En resumen, las últimas novedades y mejoras en Kubernetes ofrecen una variedad de características y herramientas para mejorar la seguridad, el rendimiento y la gestión de los clusters. Los administradores de clusters deben estar al tanto de estas novedades y planificar su implementación para aprovechar al máximo las capacidades de Kubernetes.
