Novedades en Kubernetes 1.36

2026-04-30 1010 · Kubernetes

La versión 1.36 de Kubernetes es una de las más importantes en los últimos tiempos, ya que incluye una serie de mejoras y características nuevas que permiten a los administradores de sistemas y desarrolladores trabajar de manera más eficiente. Una de las características más destacadas es la disponibilidad general de los espacios de nombres de usuario, que permite a los contenedores ejecutarse con permisos de usuario y grupo específicos.

Para aprovechar al máximo esta característica, es necesario configurar el parámetro `userns` en el archivo de configuración de Kubernetes, utilizando el comando `kubectl` de la siguiente manera: `kubectl config set-credentials --userns=`. Además, es importante mencionar que esta característica solo está disponible en sistemas Linux.

En cuanto a la seguridad, la versión 1.36 de Kubernetes incluye varias mejoras, como la autenticación y autorización basadas en roles, que permiten a los administradores asignar permisos específicos a los usuarios y grupos. Para configurar la autenticación y autorización, se puede utilizar el comando `kubectl create role` y `kubectl create rolebinding`, respectivamente.

La versión 1.36 de Kubernetes también incluye mejoras en el almacenamiento y las redes, como el soporte para persistent volumes y la capacidad de configurar redes de manera más flexible. Para configurar un persistent volume, se puede utilizar el comando `kubectl create -f .yaml`, donde el archivo de configuración contiene la definición del volumen persistente.

En resumen, la versión 1.36 de Kubernetes es una versión muy completa que incluye mejoras en seguridad, almacenamiento y redes, y que permite a los administradores de sistemas y desarrolladores trabajar de manera más eficiente. Para obtener más información sobre esta versión, se puede consultar la documentación oficial de Kubernetes.
