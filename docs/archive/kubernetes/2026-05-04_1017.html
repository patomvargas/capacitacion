Novedades en Kubernetes 1.36

2026-05-04 1017 · Kubernetes

La versión 1.36 de Kubernetes es una actualización importante que incluye varias características y mejoras. Una de las novedades más destacadas es la disponibilidad general de User Namespaces, que permite a los administradores de clusters aislar y gestionar los recursos de los usuarios de manera más efectiva. Para habilitar esta característica, se puede utilizar el comando `kubectl create namespace --dry-run=client -o yaml | kubectl apply -f -`.

Otra característica importante es la escalabilidad vertical en lugar para recursos de nivel de pod, que permite a los administradores ajustar los recursos de los pods sin tener que recrearlos. Esto se puede lograr utilizando el comando `kubectl scale --replicas= `. Además, la versión 1.36 incluye mejoras en la seguridad, como la autenticación y autorización basadas en roles, que se pueden configurar utilizando el comando `kubectl create role --verb= --resource=`.

En cuanto al almacenamiento, la versión 1.36 de Kubernetes incluye mejoras en el soporte para almacenamiento persistente, como el uso de StatefulSets para gestionar los datos de los pods. Para crear un StatefulSet, se puede utilizar el comando `kubectl create statefulset --image= --replicas=`. Además, la versión 1.36 incluye mejoras en la gestión de los recursos de red, como el uso de Network Policies para controlar el tráfico de red entre los pods.

La versión 1.36 de Kubernetes también incluye mejoras en la herramienta de línea de comandos `kubectl`, como la capacidad de utilizar comandos de forma más interactiva. Por ejemplo, se puede utilizar el comando `kubectl debug -- ` para depurar un pod en ejecución. Además, la versión 1.36 incluye mejoras en la documentación y la comunidad de Kubernetes, como la creación de un nuevo sitio web de documentación y la organización de eventos y webinars para la comunidad de desarrolladores.

En resumen, la versión 1.36 de Kubernetes es una actualización importante que incluye varias características y mejoras en áreas como la escalabilidad, la seguridad y el almacenamiento. Los administradores de clusters y desarrolladores pueden aprovechar estas novedades para mejorar la gestión y el desempeño de sus aplicaciones en Kubernetes.

Para obtener más información sobre la versión 1.36 de Kubernetes, se pueden consultar las fuentes oficiales de la comunidad de Kubernetes, como el sitio web de documentación y el blog de Kubernetes.
